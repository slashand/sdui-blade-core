# The Journey Architecture Manifesto
*An Expert Thinking Diary for the SDUI Blade System*

> **"A UI is not a collection of pages. It is a continuous, unfolding spatial journey."**

This repository contains the architectural thinking, historical evolution, and design documents for `@slashand/sdui-blade-core` and its React/Angular counterparts. We treat this as an "Expert Thinking Diary"—a living testament to the engineering decisions required to build a framework-agnostic, infinitely scalable horizontal orchestration system.

If you are an open-source contributor, enterprise architect, or UI engineer looking to understand the profound paradigm shift required to build a "Journey-based" interface, start here.

## Volumes

1. [**The Journey Protocol (001)**](./001-the-journey-protocol.md) - The foundational philosophy of horizontal spatiality, durable vs. transient state, and the end of the "Modal" era.
2. [**The Inversion of Mount Points (002)**](./002-inversion-of-mount-points.md) - How the agnostic core engine provisions empty space for independent React/Angular micro-frontends to inhabit.
3. [**The Taxonomic Hierarchy (003)**](./003-the-taxonomic-hierarchy.md) - Understanding Journeys, Blades, Lenses, Parts, and Command Bars.
4. [**The Styling Constants (004)**](./004-the-styling-constants.md) - The mathematical CSS variable contract.
5. [**Schema and Types (005)**](./005-schema-and-types.md) - Deep dive into the `SduiBladeNode` JSON schema and elements.
6. [**Spatial Boundaries and Regions (006)**](./006-spatial-boundaries-and-regions.md) - Defining layout targets for Blade execution.
